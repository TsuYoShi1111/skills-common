# 業務マニュアル作成

業務マニュアル・手順書を、誰が読んでも同じ結果を再現できる品質で作成する。

## ワークフロー

### Step 1: ヒアリング

1. **マニュアルの種類**
   - 業務手順書（特定の業務の進め方）
   - ツール操作ガイド（ソフトウェア・サービスの使い方）
   - 新人研修マニュアル（入社時に読むもの）
   - 引き継ぎ資料（担当変更時の情報まとめ）
2. **対象読者**（新人/中途/外部パートナー）
3. **対象業務の概要**
4. **前提知識**（読者が知っている想定の範囲）

### Step 2: 構成案の提示

目次レベルの構成を提示し、承認を得る。

### Step 3: 本文作成

承認された構成に沿って作成する。

## テンプレート構成

### 業務手順書

```
# 【業務名】マニュアル

## 概要
- 目的：この業務で達成すること
- 対象者：誰がこの業務を行うか
- 頻度：日次 / 週次 / 月次 / 都度
- 所要時間：約◯分

## 事前準備
- [ ] 必要なツール・アカウント
- [ ] 必要な権限
- [ ] 参照する資料

## 手順

### Step 1: 【作業名】
**目的**: このステップで何を達成するか

1. 【具体的な操作1】
2. 【具体的な操作2】

> 注意: 【間違えやすいポイント】

✅ 完了条件: 【この状態になっていればOK】

### Step 2: 【作業名】
（同じ形式で繰り返す）

## トラブルシューティング

| 症状 | 原因 | 対処法 |
|------|------|--------|
| | | |

## 関連資料
- 【リンクや参照先】

## 更新履歴
| 日付 | 更新者 | 内容 |
|------|--------|------|
```

## 作成ルール

- 1ステップ1アクション（複数の操作を1つにまとめない）
- 画面キャプチャの挿入位置を【スクリーンショット：◯◯画面】で指示
- 判断が必要な場面は「〜の場合は→Aへ」の分岐で明示
- 専門用語には初出時に（）で説明を添える
- 完了条件を各ステップに記載（「この状態になっていればOK」）
- NG例・よくあるミスも記載する
- 数値や固定値は太字で強調
