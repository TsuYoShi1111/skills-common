# Competitive Analysis

競合Webサイトを多角的に分析し、差別化戦略と改善案をレポートする。

## ワークフロー

### Step 1: ヒアリング

以下を確認する：

1. **自社情報**
   - 業種・サービス内容
   - ターゲット顧客
   - 自社サイトURL（あれば）
   - 強み・差別化ポイント（自認しているもの）

2. **競合情報**
   - 分析したい競合サイトURL（2〜5社推奨）
   - 競合として認識している理由
   - 「この競合のここが良い」と思うポイント（あれば）

3. **分析の目的**
   - 新規サイト制作の方向性決め
   - 既存サイトのリニューアル参考
   - 特定の要素（デザイン / コンテンツ / SEO）の改善

### Step 2: 分析実施

各競合サイトを以下の5カテゴリで分析する。

#### カテゴリ1: サイト構成・IA（情報設計）

| 分析項目 | 確認ポイント |
|---------|------------|
| ページ構成 | どんなページがあるか、階層構造 |
| ナビゲーション | メニュー項目、グローバルナビ、フッターナビ |
| CTA配置 | どこにCTAがあるか、何を促しているか |
| コンバージョン導線 | 訪問→問い合わせまでの流れ |
| サイトマップ | 全体のページ数とカテゴリ構造 |

#### カテゴリ2: デザイン・ビジュアル

| 分析項目 | 確認ポイント |
|---------|------------|
| カラースキーム | メイン・サブ・アクセント色 |
| タイポグラフィ | フォント種類、サイズ感 |
| ファーストビュー | キャッチコピー、ビジュアル、CTA |
| レイアウト | 余白の使い方、グリッド構造 |
| 画像・イラスト | 写真の品質、イラストの雰囲気 |
| レスポンシブ | モバイル対応の品質 |

#### カテゴリ3: コンテンツ戦略

| 分析項目 | 確認ポイント |
|---------|------------|
| キャッチコピー | メインメッセージ、訴求ポイント |
| サービス説明 | 説明の仕方、ベネフィットの訴求方法 |
| 社会的証明 | お客様の声、実績数値、メディア掲載 |
| ブログ・コラム | 記事の頻度、テーマ、品質 |
| 差別化メッセージ | 「なぜウチか」の伝え方 |

#### カテゴリ4: SEO対策

| 分析項目 | 確認ポイント |
|---------|------------|
| titleタグ | 各ページのtitle内容とキーワード |
| meta description | 記述の有無と内容 |
| 見出し構造 | h1〜h3の使い方 |
| 構造化データ | JSON-LDの有無と種類 |
| コンテンツ量 | テキスト量、ページ数 |

#### カテゴリ5: UX・機能

| 分析項目 | 確認ポイント |
|---------|------------|
| 表示速度 | 体感速度、画像の最適化具合 |
| フォーム | 項目数、入力のしやすさ |
| インタラクション | アニメーション、ホバー効果 |
| 独自機能 | 料金シミュレーター、チャット等 |
| SNS連携 | SNSリンク、シェアボタン |

### Step 3: 比較表の作成

全競合を一覧で比較する：

```

|------|------|-------|-------|-------|
| ファーストビュー | - | 動画BG | 静止画 | スライダー |
| CTA | - | 無料相談 | 資料DL | LINE登録 |
| 色 | - | 青系 | 緑系 | 紺系 |
| ページ数 | - | 12 | 8 | 15 |
| ブログ | - | 月4本 | なし | 月2本 |
| お客様の声 | - | 6件 | 3件 | 10件 |
| 構造化データ | - | あり | なし | あり |
...
```

### Step 5: 次のアクションの提案

分析結果に基づき、具体的な次のステップを提案：

- サイト制作なら → site-builder スキルでプロジェクト開始
- コンテンツ改善なら → copywriting / web-content スキルで作成
- SEO改善なら → /seo-check コマンドで監査
